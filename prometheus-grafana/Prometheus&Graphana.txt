Разные namespaces - monitoring (для Prometheus, Graphana & NGINX Ingress) и otusapp соответственно.

Для сбора / публикации метрик использовал ASP.NET Core экспортер - https://github.com/prometheus-net/prometheus-net.

Приложение задеплоено на два инстанса / пода.

Нагружал / собирал метрики для двух API-методов - Health (сразу отдаёт ответ) и GetAllProducts (возвращает продукты из БД, поэтому latency для него в 2-3 раза больше).
Для генерации нагрузки через Apache Benchmark (AB) в цикле отправлял запросы то к одному методу, то к другому (5000 запросов в 300 и 100 потоков соответственно).
 - ab.exe -n 5000 -c 300 http://arch.homework/products
 - ab.exe -n 5000 -c 100 http://arch.homework/products/health

Не получилось вызвать ошибки, увеличивая нагрузку, поэтому искусственно каждый 5й запрос к каждому методу сразу генерирует 500 Internal Server Error.
